# Code Review

General code review checklist.

## Checklist

1. **Tests**: Verify tests exist for all changed code
2. **Lint**: Run the project linter and fix all warnings
3. **Error Handling**: All operations have proper error handling
4. **Naming**: Follow project naming conventions
5. **Security**: No hardcoded secrets, validate user input
6. **Documentation**: Public APIs are documented
7. **Performance**: No obvious bottlenecks or resource leaks

## How to use
Run `/code-review` after completing implementation to verify quality.
